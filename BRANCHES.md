# Branch Guide

This repository uses a small branch model for contributors.

## Protected Branches

- `master`: stable branch. Pull requests only.
- `develop`: integration branch. Pull requests only.

## Contributor Branches

- `feature/<name>`: new editor/runtime features.
- `fix/<name>`: bug fixes.
- `docs/<name>`: documentation updates.
- `examples/<name>`: sample projects and templates.

## Long-Lived Helper Branches

- `develop`: integration branch for code changes.
- `docs`: base branch for documentation-heavy contributions.
- `examples`: base branch for starter projects and sample scenes.

When in doubt, open a draft pull request early and ask for feedback.
