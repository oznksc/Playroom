# Governance

Playroom is maintained by the repository owner and trusted maintainers.

## Decision Making

Maintainers prioritize:

1. A reliable MVP editor/runtime workflow.
2. Clear schema compatibility.
3. Small, reviewable changes.
4. Documentation that helps contributors build and test locally.

Large scope changes should start as an issue or discussion before code is
written. Maintainers may close work that expands the product beyond the current
MVP without a clear migration path.

## Maintainer Responsibilities

Maintainers are responsible for:

- Reviewing pull requests.
- Keeping `master` stable.
- Labeling issues for contributors.
- Enforcing the Code of Conduct.
- Coordinating security fixes.

## Contributor Path

Regular contributors can earn additional trust by submitting focused pull
requests, reviewing issues, improving tests, and documenting reproducible
workflows.
